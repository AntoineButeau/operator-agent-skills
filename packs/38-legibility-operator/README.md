# Legibility Operator Pack

## Source series
Legibility and Illegibility for Operators

## Who it is for
Operators, founders, executives, managers, chiefs of staff, BizOps leads, product leaders, revenue leaders, and AI agents helping them make decisions from live operating context.

## When to use this pack
Use this pack to convert legibility and illegibility for operators material into operator-grade decisions and reusable artifacts. The four skills are deliberately sequenced so diagnosis, mapping, planning, and review do not collapse into one vague prompt.

## Progression / use order
1. [Legibility Audit](skills/legibility-audit.md) — diagnose the current state and name the operating risk.
2. [Strategic Illegibility Check](skills/strategic-illegibility-check.md) — diagnose the current state and name the operating risk.
3. [Visibility System Design](skills/visibility-system-design.md) — produce a reusable artifact or operating spec.
4. [Legible Decision Package](skills/legible-decision-package.md) — handle the package step without overlapping the other skills in the pack.
