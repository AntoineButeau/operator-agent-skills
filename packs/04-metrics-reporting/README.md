# Decision Metrics Pack

## Source series
Goals / Metrics / Dashboards / Reporting

## Who it is for
Operators, founders, executives, managers, chiefs of staff, BizOps leads, product leaders, revenue leaders, and AI agents helping them make decisions from live operating context.

## When to use this pack
Use this pack to turn metric sprawl into decision dashboards, owner maps, and operating-review signals. The four skills are deliberately sequenced so diagnosis, mapping, planning, and review do not collapse into one vague prompt.

## Progression / use order
1. [Metric Quality Audit](skills/metric-quality-audit.md) — diagnose the current state and name the operating risk.
2. [Decision Dashboard Brief](skills/decision-dashboard-brief.md) — compress the situation into a decision-ready brief.
3. [Weekly Report Upgrade](skills/weekly-report-upgrade.md) — handle the upgrade step without overlapping the other skills in the pack.
4. [Metric Owner Map](skills/metric-owner-map.md) — make the system visible by mapping owners, flows, and evidence.
