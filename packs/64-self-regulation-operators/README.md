# Self-Regulation Operator Pack

## Source series
Self-Regulation for Operators

## Who it is for
Operators, founders, executives, managers, chiefs of staff, BizOps leads, product leaders, revenue leaders, and AI agents helping them make decisions from live operating context.

## When to use this pack
Use this pack to convert self-regulation for operators material into operator-grade decisions and reusable artifacts. The four skills are deliberately sequenced so diagnosis, mapping, planning, and review do not collapse into one vague prompt.

## Progression / use order
1. [Trigger Pattern Map](skills/trigger-pattern-map.md) — make the system visible by mapping owners, flows, and evidence.
2. [Response Gap Plan](skills/response-gap-plan.md) — turn the diagnosis into an executable plan with owners and gates.
3. [Emotional Load Review](skills/emotional-load-review.md) — diagnose the current state and name the operating risk.
4. [Operator Reset Protocol](skills/operator-reset-protocol.md) — define a repeatable protocol with safe human checkpoints.
