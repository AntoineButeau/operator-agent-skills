# COO Operating Model Pack

## Source series
How to Be a Great COO: The Operating System Behind the Company

## Who it is for
Operators, founders, executives, managers, chiefs of staff, BizOps leads, product leaders, revenue leaders, and AI agents helping them make decisions from live operating context.

## When to use this pack
Use this pack to convert how to be a great coo: the operating system behind the company material into operator-grade decisions and reusable artifacts. The four skills are deliberately sequenced so diagnosis, mapping, planning, and review do not collapse into one vague prompt.

## Progression / use order
1. [Company Operating Model Map](skills/company-operating-model-map.md) — make the system visible by mapping owners, flows, and evidence.
2. [Cross-Functional Issue Room](skills/cross-functional-issue-room.md) — handle the room step without overlapping the other skills in the pack.
3. [Execution Risk Register](skills/execution-risk-register.md) — handle the register step without overlapping the other skills in the pack.
4. [COO Operating Dashboard](skills/coo-leverage-dashboard.md) — handle the dashboard step without overlapping the other skills in the pack.
