# Org Design Diagnostic Pack

## Source series
Organizational Design

## Who it is for
Operators, founders, executives, managers, chiefs of staff, BizOps leads, product leaders, revenue leaders, and AI agents helping them make decisions from live operating context.

## When to use this pack
Use this pack to convert organizational design material into operator-grade decisions and reusable artifacts. The four skills are deliberately sequenced so diagnosis, mapping, planning, and review do not collapse into one vague prompt.

## Progression / use order
1. [Org Design Diagnosis](skills/org-design-diagnosis.md) — diagnose the current state and name the operating risk.
2. [Decision Rights Matrix](skills/decision-rights-matrix.md) — handle the matrix step without overlapping the other skills in the pack.
3. [Handoff Friction Map](skills/handoff-friction-map.md) — make the system visible by mapping owners, flows, and evidence.
4. [Role Clarity Redesign](skills/role-clarity-redesign.md) — produce a reusable artifact or operating spec.
