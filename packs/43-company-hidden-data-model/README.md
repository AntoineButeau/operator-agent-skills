# Hidden Data Model Pack

## Source series
The Company’s Hidden Data Model

## Who it is for
Operators, founders, executives, managers, chiefs of staff, BizOps leads, product leaders, revenue leaders, and AI agents helping them make decisions from live operating context.

## When to use this pack
Use this pack to convert the company’s hidden data model material into operator-grade decisions and reusable artifacts. The four skills are deliberately sequenced so diagnosis, mapping, planning, and review do not collapse into one vague prompt.

## Progression / use order
1. [Hidden Data Model Map](skills/hidden-data-model-map.md) — make the system visible by mapping owners, flows, and evidence.
2. [Entity Ownership Audit](skills/entity-ownership-audit.md) — find gaps between stated process and observed behavior.
3. [Shadow Data Reconciliation Plan](skills/shadow-data-reconciliation-plan.md) — turn the diagnosis into an executable plan with owners and gates.
4. [Decision Data Fit Review](skills/decision-data-fit-review.md) — diagnose the current state and name the operating risk.
