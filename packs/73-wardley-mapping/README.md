# Wardley Mapping Pack

## Source series
Wardley Mapping for Operators: How to See the Strategic Landscape

## Who it is for
Operators, founders, executives, product leaders, platform leads, BizOps teams, AI transformation owners, and agents helping them make resource, platform, vendor, staffing, and operating-model decisions from a visible strategic landscape.

## When to use this pack
Use this pack when a team needs situational awareness before choosing a strategy. The four skills move from user-anchored landscape mapping to dependency analysis, evolution-stage fit, and doctrine/gameplay decisions. They are designed for messy operating questions where org charts, roadmap preferences, vendor pressure, or AI hype are hiding the actual map.

## Progression / use order
1. [Strategic Landscape Map Builder](skills/strategic-landscape-map-builder.md) — anchor the map in the user need, name the purpose, and sketch the strategic landscape before decisions harden.
2. [Value Chain Dependency Mapper](skills/value-chain-dependency-mapper.md) — expose the capabilities, data, tools, vendors, models, people, and governance the user outcome depends on.
3. [Evolution Operating Model Fit Check](skills/evolution-operating-model-fit-check.md) — test whether each capability is being run with methods that fit its evolution stage.
4. [Doctrine Gameplay Decision Brief](skills/doctrine-gameplay-decision-brief.md) — separate basic doctrine fixes from strategic moves such as build, buy, commoditize, differentiate, partner, constrain, or exit.
