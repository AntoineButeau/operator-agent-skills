# Harness Engineering Pack

## Source series
Harness Engineering: The Runtime Layer Around AI Agents

## Who it is for
Operators, founders, executives, managers, chiefs of staff, BizOps leads, product leaders, revenue leaders, and AI agents helping them make decisions from live operating context.

## When to use this pack
Use this pack to convert harness engineering: the runtime layer around ai agents material into operator-grade decisions and reusable artifacts. The four skills are deliberately sequenced so diagnosis, mapping, planning, and review do not collapse into one vague prompt.

## Progression / use order
1. [Agent Runtime Architecture Map](skills/agent-runtime-architecture-map.md) — make the system visible by mapping owners, flows, and evidence.
2. [Eval Harness Spec](skills/eval-harness-spec.md) — handle the system step without overlapping the other skills in the pack.
3. [Context Window Management Plan](skills/context-window-management-plan.md) — turn the diagnosis into an executable plan with owners and gates.
4. [Runtime Failure Mode Review](skills/runtime-failure-mode-review.md) — diagnose the current state and name the operating risk.
