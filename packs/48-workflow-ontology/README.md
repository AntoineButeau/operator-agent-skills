# Workflow Ontology Pack

## Source series
Workflow Ontology

## Who it is for
Operators, founders, executives, managers, chiefs of staff, BizOps leads, product leaders, revenue leaders, and AI agents helping them make decisions from live operating context.

## When to use this pack
Use this pack to convert workflow ontology material into operator-grade decisions and reusable artifacts. The four skills are deliberately sequenced so diagnosis, mapping, planning, and review do not collapse into one vague prompt.

## Progression / use order
1. [Workflow Entity Map](skills/workflow-entity-map.md) — make the system visible by mapping owners, flows, and evidence.
2. [State Transition Audit](skills/state-transition-audit.md) — find gaps between stated process and observed behavior.
3. [Work Object Taxonomy](skills/work-object-taxonomy.md) — handle the system step without overlapping the other skills in the pack.
4. [Ontology Change Control](skills/ontology-change-control.md) — handle the system step without overlapping the other skills in the pack.
