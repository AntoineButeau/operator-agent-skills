# Mastery Operating System Pack

## Source series
Mastery Is an Operating System

## Who it is for
Operators, founders, executives, managers, chiefs of staff, BizOps leads, product leaders, revenue leaders, and AI agents helping them make decisions from live operating context.

## When to use this pack
Use this pack to convert mastery is an operating system material into operator-grade decisions and reusable artifacts. The four skills are deliberately sequenced so diagnosis, mapping, planning, and review do not collapse into one vague prompt.

## Progression / use order
1. [Mastery Loop Map](skills/mastery-loop-map.md) — make the system visible by mapping owners, flows, and evidence.
2. [Deliberate Practice Plan](skills/deliberate-practice-plan.md) — turn the diagnosis into an executable plan with owners and gates.
3. [Feedback Quality Review](skills/feedback-quality-review.md) — diagnose the current state and name the operating risk.
4. [Skill Progression Scorecard](skills/skill-progression-scorecard.md) — define evaluation criteria and evidence standards.
