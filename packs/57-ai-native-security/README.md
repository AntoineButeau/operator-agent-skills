# AI-Native Security Pack

## Source series
The AI-Native Security Model

## Who it is for
Operators, founders, executives, managers, chiefs of staff, BizOps leads, product leaders, revenue leaders, and AI agents helping them make decisions from live operating context.

## When to use this pack
Use this pack to convert the ai-native security model material into operator-grade decisions and reusable artifacts. The four skills are deliberately sequenced so diagnosis, mapping, planning, and review do not collapse into one vague prompt.

## Progression / use order
1. [Agent Threat Model](skills/agent-threat-model.md) — handle the system step without overlapping the other skills in the pack.
2. [Data Permission Review](skills/data-permission-review.md) — diagnose the current state and name the operating risk.
3. [AI Security Control Map](skills/ai-security-control-map.md) — make the system visible by mapping owners, flows, and evidence.
4. [Red-Team Findings Brief](skills/red-team-findings-brief.md) — produce a reusable decision brief or operating spec.
